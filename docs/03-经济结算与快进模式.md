# 03 · 经济结算 + 快进模式（economy + paradigm扩展）

> 模块：`src/game/economy/`（M4）+ paradigm 快进扩展（用户修改2）。纯函数，可单测。
> 设计依据：经济模型_v1.md（含§14用户修正）。

## economy 做什么
把经济模型的所有数值公式落成纯函数。供奉吞吐/避孕套/欲望滚雪球/武力/招募/据点战胜率。
常数集中在 `CONST`，便于数值调平。

## 核心公式（machine.ts）
| 函数 | 公式 | 备注 |
|------|------|------|
| nightCoverage(格,吞吐) | 格×吞吐 | 一晚供奉覆盖人数 |
| slotsNeededFor(打手,吞吐) | ceil(打手/吞吐) | 覆盖所需夜晚格 |
| unservedCount | max(0,打手-覆盖) | 未供奉缺口→欲望 |
| condomCost(人数,危险期) | 人数×3 ×(危险期1.5) | 避孕套消耗 |
| purchaseCap(店铺,稳定,升级) | 360×店铺×稳定×升级 | 单次采购上限 |
| desireGain(未供奉,连续3晚数) | 普通×2 + 连续×2×2 | 滚雪球 |
| combatPower(可用打手,忠诚) | 可用×(忠诚/100) | 忠诚低武力打折 |
| weeklyRecruitQuota(总威望) | 5+威望×0.5 | 每周招募额度 |
| siegeWinChance | 武力比+威望差+情报,夹2%~98% | 据点战胜率 |

## 吞吐=可升级成长项（用户修改1·重要）
- **每格吞吐不再固定18**，schema 新增 `时间.每格供奉吞吐`，初始 **6**，可升 12/18/24/30。
- 升级来源：设施升级 + 堕落度解锁更高档（堕落易）。
- 影响：初期吞吐低(6)，30打手需5夜晚格(留3白天)→初期就有压力；玩家靠"升吞吐+加格"双线提处理力；**百人甜区是动态的**(吞吐升级 vs 打手增速赛跑)，不再是固定门槛。
- 测试验证：slotsNeededFor(100,6)=17(初期吃紧)，吞吐升到18才=6格覆盖百人(旧甜区)。

## 快进模式（用户修改2·paradigm扩展）
- 行动格增"快进开关"。`renderModeFor(pick, fastForward)` 决定渲染方式：
  - 首次特殊事件 → **永远 `ai_full`**（堕落里程碑不跳过，即使快进开着）。
  - 非首次 + 快进 → `fast_summary`：**不调AI**，系统输出 CG + 总结词("大小姐被{n}人插入了")+直接更新变量。
  - 非首次 + 非快进 → `ai_brief`（略写）。
- `fastSummaryText(模板, vars)` 填充总结词。CG 来自自制美术(comfyui,后期)。
- 价值：色情玩家跳过日常流水直接刷数值/看CG；经营玩家关快进享受叙事。
