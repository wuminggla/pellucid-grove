# 13 · 叙事管线与记忆层（阶段3 实现）

> 落地设计正典 §13b 三阶段设计(范式约束力边界/记忆层/prompt组装管线)。
> 解决两大架构缺口:① AI失忆(程序主导叙事,酒馆楼层补不全) ② 生态没接(世界书/预设/JB全绕开)。
> 模块:`src/game/worldbook/` `src/game/memory/` `src/game/transcript/` + `engine/prompt.ts`(buildGamePrompt)。

## 一、世界书/预设接入（worldbook·阶段3-1）
- 世界书=内容仓库(复用底座 Lorebook)。`getConstantEntries`(constant蓝灯永久注入:美学/叙事/角色/打手态度)、`getParadigmByKey`(代码按key直取范式正文·不走关键词scan·防胡诌)、`renderConstantBlock`。
- demo世界书/预设为占位,真实内容随"内容填充"填。

## 二、buildGamePrompt（prompt.ts·阶段3-2）
- 代码驱动组装,**不照搬关键词驱动的 assemblePrompt**,复用其零件(replaceMacros)。
- 组装序:预设main/JB → 常驻世界书 → [故事脉络] → [近期发生] → 本格范式(按key/inlinePrompt) → 认知态度层 → 当前场景 → 扩写规格 → 输出格式。
- `presetSampling` 取采样参数透传 api-router。real-ai 注入 buildGamePrompt + sampling。
- **JB越狱块**:NSFW过审关键,经预设注入(之前完全缺失)。

## 三、记忆层（memory·阶段3-3 / 3-3b）
> 范式锁住"孤立这场戏"(桶1/桶2),锁不住跨事件连续性(桶4)和自由血肉(桶3)。记忆层只装桶2+桶4。
- **结构化日志(桶2)**:`narrativeLog`,day-runner 每格代码写(覆盖快进/略写/纯代码格,不依赖AI)。`renderRecentLog` 滑动窗口最近12条 → [近期发生]。
- **延续摘要(桶4)**:`continuityNotes` 永久,渲染→[故事脉络]。两条来源:
  - 代码笔记:认知防线跨档(turning)、首次特殊兜底(milestone)。
  - **AI piggyback(3-3b)**:`needsContinuity` 事件→AI1 同次调用多吐 `<continuity>` 一句(real-ai `extractContinuity` 解析)→entity笔记(优先于首次兜底)。`AiPort.expand` 返回 `{text, continuity}`。
- `needsContinuity = EventOption.needsContinuity || 认知跨档`。
- 不装:桶1(范式自带)、桶3(→正文留档)、可由结构化事实重建的。
- 随存档持久化(EngineState.narrativeLog/continuityNotes)。

## 四、正文留档（transcript·阶段3-4）
- 桶3去处:AI全文正文(maintext)存独立IndexedDB(`JiutiaoTranscriptDB`,与存档/底座隔离),给玩家翻阅画廊/日记,**不进AI记忆**。
- `archiveTranscript`(空正文跳过)/`listTranscripts`(时间序)/`getTranscript`/`deleteTranscript`(可删)/`clearTranscripts`。带元数据(day/事件/在场/认知/堕落/renderMode/isNsfw)。
- 自动留档:`useDayRunner.runCurrent` 每格执行后 fire-and-forget 留档(失败不阻塞)。day-runner 返回 `logEntry` 供复用。

## 分工边界(接回生态后)
- 选范式/首次/renderMode = 代码(防胡诌)。范式/角色/美学内容 = 生态世界书。采样/JB/宏 = 生态预设。最终拼装 = 我们 buildGamePrompt。故事记忆 = 我们 memory。正文画廊 = 我们 transcript。
