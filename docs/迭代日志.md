| 版 | 问题 | 解决 |

| v05 | 改用 `$('#mount').load(URL)` 模式·状态栏正则输出 mount div + script | jQuery.ajax 被酒馆全局加了 `x-csrf-token` header,跨域时 CORS preflight 失败被浏览器阻止 |

**设计思路·最终架构**:
```
酒馆角色卡 (九条会-pellucid-v10.png)
  └─ extensions.tavern_helper.scripts.PellucidLoader
        ├─ id: pellucid-loader-v10-mqvg5d9j  (每次大改换新 id 绕酒馆助手脚本缓存)
        ├─ enabled: true
        └─ content: import 'https://testingcf.jsdelivr.net/gh/wuminggla/pellucid-grove@2201a0e/dist/jiutiao/脚本/PellucidLoader/index.js';
                              ↑ commit sha 锁版本·jsdelivr 永久缓存且不会跟其他 commit 串味

远程 ESM (dist/jiutiao/脚本/PellucidLoader/index.js · 由 webpack 编译输出 · src/jiutiao/脚本/PellucidLoader/index.ts 是源码)
  在 TH-script--PellucidLoader--<id> 隐藏 iframe 里执行
  ↓ 用 window.top.document.body 跳出 sandbox
  在主页面 body 创建可见浮动容器 #pellucid-app-root (fixed right top 460×640px)
  ↓ fetch dist/jiutiao/界面/状态栏/index.html (Vue webpack 编译的单文件 HTML,JS/CSS 内联)
  patchHtml() 在 <head> 注入 jQuery + Vue UMD (远程 HTML 用 const o=Vue / $() 依赖全局)
  ↓ 把 patched html 作为 srcdoc 注入容器内的新 iframe (JS 动态创建,绕过 sanitize)
  iframe 内独立执行 script → Vue mount #app → 渲染 App.vue (当前 = 空壳标题占位)
```

**关键技术点·下次接手必读**:
1. **酒馆助手脚本沙箱机制**: 每个卡脚本放在隐藏 iframe(`TH-script--<name>--<id>`)·内部 `document.body` 不是主页面 body·要操作主页 DOM 必须用 `window.top.document`
2. **sanitize 黑名单**: 状态栏正则 replaceString 输出的 `<iframe>` / `<script>` 标签会被酒馆 sanitize 剥(`<script>` 内容会被当文本暴露).**JS 动态 createElement 创建的 iframe 不受 sanitize 影响**——这是注入 Vue 的合法路径
3. **jsdelivr 缓存反直觉**: `?v=xxx` query string cache buster 在 jsdelivr 边缘节点**不一定生效**.可靠方案是用 commit sha 锁 URL(`@<sha7>/...`)·jsdelivr 视为不可变资源
4. **content 字段标准模式**: 酒馆助手按 ES module 严格模式执行 content·**不能塞 IIFE 源码**(全局 `$`/`Vue` 不可用 → ReferenceError 被静默吞).标准是 `content="import 'jsdelivr_URL'"` 一行·真实逻辑在 webpack 编译后的远程 ESM 里
5. **远程 HTML 的全局依赖**: webpack 编译 Vue 时把 `vue` 当 external 全局(`const o = Vue`).srcdoc iframe 没有 Vue 全局,所以 loader 要 patchHtml() 在 head 注入 jQuery + Vue UMD

**接口/扩展点**:
- `src/jiutiao/脚本/PellucidLoader/index.ts` — loader 源码·改 srcdoc iframe 行为/容器样式/位置都在这里
- `src/jiutiao/界面/状态栏/App.vue` — Vue 主组件·下次 UI 翻译的入口
- 卡里 `jiutiao/脚本/PellucidLoader.txt` — 一行 import URL·**每次推送新远程 loader 后要更新 sha**
- 流程: 改 src → `pnpm build` → `git push` 推到 master → `git rev-parse HEAD | cut -c1-7` 取新 sha → 改 PellucidLoader.txt URL → 改 state.json (PellucidLoader.id + projectName + cardrc artifact) → `pack` → 拖新卡进酒馆

**已知限制 / 待补**:
- **Vue 看不到 chat 上下文**: srcdoc iframe 跟主页面是不同的 origin/window,Vue 没法直接读酒馆 MVU 变量或调酒馆 LLM.下次要做的 `defineMvuDataStore` 集成需要 postMessage 桥或直接走主页面 window 全局.
- **每条 AI 消息都触发状态栏正则**: 但 loader 的 `loadPromise` 单例机制只 fetch 一次,后续消息渲染只会在消息内显示"前端已激活·见浮动面板"指示器,不会重复创建容器.
- **commit sha 锁定意味着发新版要更新卡**: 仅改 dist 不重打卡不会让玩家看到新代码(浏览器/jsdelivr 缓存指向旧 sha).改远程 dist 后必须 `pack` 新版卡并让玩家重拖.或者保留 `@latest`/`?v=`-style URL 用于开发期,正式发布锁 sha.

**试玩关注**:

- "前端正在初始化…"占位 = Vue 在 srcdoc iframe 内正确 mount
- 点"▁"收起按钮折叠 = jQuery 在 srcdoc 内可用

## 2026-06-27 · v11/v12 推翻 loader 路线·改用原生前端界面机制·数据层全打通 (commit bffd791)

**关键认知翻转**:
- 酒馆助手两种产物: **前端界面**(界面/xxx/index.ts + index.html · 无沙盒 iframe 前台显示 · 酒馆全局开箱即用) vs **脚本**(脚本/xxx/index.ts · 隐藏后台 iframe)。

- v04 的 CORS 错根因是"路线错(用脚本)",不是 `$('body').load` 本身有问题——前端界面在楼层 iframe 上下文执行不带主页 csrf header。

**改动**:
- 界面/状态栏/index.ts → 标准 createApp().use(createPinia()).mount('#app')
- 界面/状态栏/global.css → 九条会暗金配色变量

- 删 PellucidLoader 脚本注册 + 卡内 txt(留档 .unused-loader-route)
- 卡脚本现在只剩 MVU

**实战验证(v12 卡)**: 探测器五项全绿——
