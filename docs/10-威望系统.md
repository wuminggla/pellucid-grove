# 10 · 威望系统（prestige）

> 落地 findings 威望·定稿。威望 = 极道威望 + 淫名，决定招募难易/应聘人数；占比决定游戏风味。
> 模块：`src/game/prestige/`。增益来源=事件（EventOption 奖励字段）；消费点=招募 + 硬失败审核。

## 核心结构

- **极道威望(martialPrestige)**：累计值，靠打架/火并/复仇胜利。
- **淫名(infamy)**：累计值，靠拍AV/轮奸规模/肉体名气。
- **总威望 = 极道威望 +（淫名，仅 AV 解锁后计入）**。用户定：淫名是 AV 解锁后才引入的机制，解锁前威望只等于极道威望。
- AV 解锁标记：`EngineState.unlocked['av']`，`prestige.isAvUnlocked()` 判定。

## prestige/machine.ts

- `AV_UNLOCK_KEY` / `isAvUnlocked(unlocked)`：淫名闸门。
- `gainMartialPrestige(slice, amount)`：极道威望累计值 + 今日流量(`martialGainToday`)同步增长。
- `gainInfamy(infamy, amount)`：淫名累加（调用方须确保 AV 已解锁）。
- `auditMartial(martialGainToday, martialZeroStreak)`：每日硬失败审核 → `{ martialZeroStreak, hardFail }`。

## 威望来源（事件奖励）

- `EventOption.martialReward?` / `infamyReward?`：每次结算给（非首次限定）。
- `settleSlot` 应用：`martialReward` → `gainMartialPrestige`；`infamyReward` → 仅 `isAvUnlocked` 时 `gainInfamy`。
- 进账量回传 `SettleEvents.martialGain` / `infamyGain` 供 UI 提示。
- 真实来源后续接：火并/据点战/复仇胜利给极道威望；AV/轮奸规模给淫名。demo 暂给 `attack.martialReward=5` 使循环可跑。

## 招募真实化（economy + settlement）

- `totalPrestige(martial, infamy, avUnlocked)`：淫名仅 AV 解锁后计入。
- `settleDaily` 改为**从 state 读真实威望**（拔掉旧 `0,0` 占位），每周刷新 `recruitQuota = weeklyRecruitQuota(totalPrestige(...))`。

## 硬失败·极道威望连续2次0（settlement）

- 定稿：**极道威望分量连续2次每日审核进账为0 → 硬失败**（不是总威望）。
- 含义：只惩罚不靠实力也不靠人海打据点的纯摆烂者/A面崩盘者；纯堕落玩家靠人海推据点打赢即涨极道威望→不触发→顺滑滑向黑道女王结局。
- 实现：`martialGainToday`（流量，每日审核后重置）+ `martialZeroStreak`（连续零进账次数）。`settleDaily` 调 `auditMartial`，`hardFail = streak≥2 || money<0`(资金兜底)。
- hook 暴露 `hardFail` 信号。

## 测试（132测，prestige相关约12）

- prestige：AV闸门 / 极道威望累计+流量 / 淫名累加 / 审核连续2次0。
- economy：totalPrestige AV门控。
- engine：martialReward进账(events+state) / infamyReward受AV门控(未解锁不进/解锁后进)。
- settlement：招募AV门控 / 硬失败连续2次0+重置 / 有进账归零。
