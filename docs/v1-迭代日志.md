# 九条会前端 · 迭代日志（活档案）

> **这份文件的定位**：每次代码更新后,人话版记录"为什么这么改 + 留了什么接口 + 已知限制 + 试玩关注点"。**新对话接手必读**——比 commit message 长比 docs/NN 短,只装"设计意图"和"接口契约"。
> **维护规范**：每次有意义的更新追加一段(模板见 §0)。被改/删的设计也保留并标【废弃】,不直接删,因为旧条目记录了"为什么走过这条路"。
> **何时不写**：纯重构/测试适配/类型修正不写;有设计判断的都写。
> **与 docs/NN-*.md 的关系**：NN 是"模块说明"(架构/数据流/类型),本文件是"思路时间线"(为什么这样设计/留了哪些扩展点)。两者互补,不重复。

## 2026-06-26 · 3-5e-2 扩张日常 11 场景 (commit a31da21)

**做了什么**: mock-ai.ts 注册 11 个扩张日常 EventOption(amusement/beach/mall/camping/hiking/street/festival/concert/garden_dog/garden_rock/ancestor),按 unlockRequires 挂占据规模解锁链;demo.ts 补 22+条范式正文(各场景常规+首次+部分SFW共用 wb_garden_sfw)。

**设计思路**:
- 公众场景一律按用户已确认的"白日宣淫·真空地带"模型写——打手仗极道身份制造真空,非偷偷摸摸怕被发现。羞耻点 = 地位倒错,不是社死。这条铁律比所有场景特化更重要,写进每条范式正文的"羞耻转化"段。
- 占据规模解锁链做成"unlock 字典 key"模式(`occupy_street`/`occupy_district`/`occupy_hill`/`occupy_halfcity`/`courtyard_unlocked`),沿用 dine 已有约定。这些 key 在 EngineState.unlocked 字典里;由 turf 系统的 defeatRegion / upgrade 系统的 expand_turf 等设置——目前 turf 系统已实现区域解锁,但**那边设置的是 regions 字段,不是 unlocked 字典**——这是个隐藏的不对齐(见下文待补)。
- 街道散步特殊处理:不挂 unlock(街道随时可去);场景上"街道空间难真空化"是事实,所以这条接近"半遮蔽"范式,与其他白日宣淫场景不同。

**接口/扩展点**:
- 占据规模解锁键(unlocked 字典): `occupy_street/district/hill/halfcity/courtyard_unlocked`。turf 系统未来需要在击败 boss/defeatRegion 时同步往 unlocked 字典写这些键(目前不写,见已知限制)。

**已知限制 / 待补**:

## 2026-06-26 · A4 日常侵蚀 机制层 + settleSlot 接入 (commits e0e924b/4b8a2f7/ca075ae)

**做了什么**:
- 机制层(A4-1): prestige.transferMartialToInfamy / EngineState.bodyDevelopment 字段 / intrusion 模块(DEVELOPMENT_LABELS/getDevelopment/advanceBodyDevelopment/A4_TRIGGER_THRESHOLDS/concealProbability/rollConcealment/applyA4)。13 单测。

- 注册表标 a4: protection / buy_condoms / school / dine 4 个白天 NSFW 事件。
- 集成单测 +4: 成功路径/失败路径/未标 a4/rng 缺省。

**设计思路**:
- A4 不是独立事件,是机制 hook——挂在 settleSlot 收尾,任何 A 面 NSFW 事件自动参与。不污染事件目录(EventOption 数不爆),"日常侵蚀"在所有 A 面 NSFW 场景统一生效。
- 随机源走 SettleOptions.rng 注入 而非内部 Math.random——保证 settleSlot 纯函数性,测试可定值,存档重演确定。缺省 0.5(中性)。
- concealProbability=0.5 + martial × 0.001(封顶 0.95):低武力≈五五开,高武力(450+)≈ 95%。鼓励玩家堆武力来盖住淫乱事件。
- 隐瞒失败的代价: 极道威望按 transferRatio(默认 0.4)转移成淫名 + 忠诚 +2(打手"共担秘密"加忠诚是双刃剑——既保住关系,又把声誉变质)。
- bodyDevelopment 用 number 0-4 而不是 schema 中文枚举: 引擎层数值比较方便(阈值/推进),hook 层映射 schema 时再翻译。

**接口/扩展点**:

- `intrusion.A4_TRIGGER_THRESHOLDS`: 调平表(口/穴=2, 肛=3, 子宫=4)。后续 A4 触发扫描器(自动插临时 NSFW 格)消费此表+ `isA4TriggerReady` 判定。**当前未实现**——身体开发度推进后没有自动触发任何东西,只是数据积累。下一步可加一个 ForcedEvent: 当某部位 ≥ 阈值时,白天有概率插入对应 NSFW 临时格。

- bodyDevelopment 只在标 a4 的事件 + developsPart 时推进。**夜晚供奉(serve_oral/vaginal/anal)没标 a4**(B 面无外人,合理),但它们也应推进对应部位——目前不会。**这是 A4-3 要补的:为夜晚 isServe 类事件加 develops 字段(不带 a4,只推进开发度)**。
