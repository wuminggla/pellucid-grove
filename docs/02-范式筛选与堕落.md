# 02 · 范式筛选 + 堕落（paradigm / corruption）

> 模块：`src/game/paradigm/` + `src/game/corruption/`。M3 产出。纯逻辑层，可单测。
> 设计依据：findings 核心定位锚①（堕落=首次特殊事件累计，硬推进认知防线，反向开奖励闸门）+ 游玩体验方向.txt。

## 这两个模块做什么
玩家在行动格选了某 optionId 后："**代码层决定用哪个范式引导 AI 扩写，并按需推进堕落**"。
- **paradigm**：选范式 + 判定是否"首次特殊事件"。
- **corruption**：首次特殊事件 → 加堕落度 → 硬推进认知防线 + 触发奖励闸门。

## paradigm（范式筛选）
### 核心：pickParadigm(registry, optionId, ctx, preferId?)
- 非特殊选项 → `daily` 范式，略写，堕落度+0。
- 特殊选项：
  - 账本未触发(首次) → `special_first`，重点扩写，堕落度+corruptionWeight。
  - 账本已触发(非首次) → `special_repeat`，退化略写，+0。
- **"是否首次"完全由代码查账本(triggeredSpecials)判定，不靠 AI**。
- 首次消费后上层调 `recordTriggered` 写账本 → 下次自动落 special_repeat = "首次范式从池排除"的代码实现。
- `unlockRequires` 控制入池：未解锁的选项不出现在 `availableParadigms`（行动格下拉只展示已解锁项）。
- **范式正文（场景/节拍/打手态度）放酒馆世界书**，前端 ParadigmDef 只存元数据 + worldbookKey（指向世界书条目，AI1扩写时注入）。注：这是工程便利（正文库复用/换皮），非内容回避。

### 文件
- `types.ts`：ParadigmKind / ParadigmDef / ParadigmContext / ParadigmPick
- `machine.ts`：pickParadigm / availableParadigms / isUnlocked / recordTriggered
- `machine.test.ts`：5 单测

## corruption（堕落）
### 核心：gainCorruption(input, gain)
- 堕落度 += gain（来自首次特殊事件）。
- `advanceCognition`：按阈值表推进认知防线，**单向不回退**（max语义）。
- 检查奖励闸门：达阈值且未领过 → fire（给钱/打手）+ 写账本（一次性）。
- 返回：新堕落度/认知档/本次推进到的档(供叙事)/触发的闸门/更新账本。

### 配置表（起调值，可调）
- 认知阈值：死撑0 / 动摇8 / 崩溃20 / 母猪化40（递增）。
- 奖励闸门：堕落度5→给钱5k+打手20；15→20k+50；30→80k+150。体现"堕落易"（越堕落资源越多）。

### 文件
- `machine.ts`：COGNITION_ORDER/THRESHOLDS/REWARD_GATES + stageForCorruption/advanceCognition/gainCorruption
- `machine.test.ts`：9 单测（阈值映射/单向/推进/闸门一次性/多闸门同跨/配置自洽）

## 调用链（上层 M5 接入时的用法）
```
玩家选 optionId → pickParadigm → 得 ParadigmPick
  → AI1 按 worldbookKey 注入范式扩写正文
  → 若 isFirstSpecial:
      gainCorruption(state, corruptionGain)
        → 更新 堕落度/认知防线/资金/打手(奖励闸门)/账本
      recordTriggered → 写 已触发特殊事件 账本
  → AI2 抓数值结算其余变量(economy, M4)
```

## 关键设计决策
1. **堕落唯一来源 = 首次特殊事件**（非"AI软主持/玩家选高风险选项软晋升"）。彻底落实核心定位锚①，杜绝参考卡软推进污染。
2. **首次判定在代码、靠账本**：可靠、不被 AI 绕过；首次后自动退化略写。
3. **认知防线单向**：advanceCognition 用 max 语义，schema 也标了单向。
4. **奖励闸门一次性**：claimedGates 账本防重复领。
5. 纯函数：易测、易回退、契合 React 不可变状态。
