# 11 · 升级系统（upgrade）

> 模块：`src/game/upgrade/`。落地 设计正典 §5。升级=独立子菜单(不占行动格)，群体整体，每级固定价。
> 数据驱动可扩展：新升级项=往 catalog append 一条 UpgradeDef，引擎只认结构，无需改代码。

## 数据模型（types.ts）

- **UpgradeDef**：`{ id, category, name, desc, cost(每级固定价), maxLevel, corruptionGate?, requires?, effect }`。
- **UpgradeCategory**：`thug`(打手·战力/NSFW) / `facility`(经营数值) / `expansion`(解锁NSFW区域与系统)。
- **UpgradeEffectKind**：`combat`(派生战力) / `throughput` / `desireCap` / `actionSlots` / `purchaseMult` / `turfFortify` / `unlock`(通用解锁,带unlockKey) / `occupyScale`(地盘扩张)。
- **UpgradeRequire**：`{ upgradeId?, minLevel?, occupyAtLeast? }` 前置依赖，全满足才出现/可买 → 体现"扩张/解锁带来新升级任务"。

## 核心逻辑（machine.ts）

- **getLevel** / **requiresMet** / **canUpgrade**(前置→满级→堕落门槛→资金 依次判定) / **applyUpgrade**(扣钱+升级+效果作用) / **combatBonus**(派生:Σ combat项等级×每级比例)。
- applyUpgrade 效果分发：combat 不写字段(派生)；其余按 kind 增量作用到状态字段(perSlotThroughput/desireCapacity/totalSlots/purchaseUpgradeMult/turfFortifyBonus/occupyScale/unlocked)。

## catalog（15项·数值为占位待调平）

**打手升级(5)**：兵器装备/格斗训练(纯战力0.10/级) · 精力强化/体型改造/调教器具(纯NSFW联想+小战力0.03/级)。卡琳典狱长式数值叙事。
**设施升级(5)**：行动格扩容(+1格/级,maxLv12) · 吞吐扩容(+6人/级,堕落解档[0,15,35,60]) · 欲望承载上限(+20/级) · 采购扩容(+0.5倍/级) · 据点加固(+1/级)。
**扩张解锁(5)**：地盘扩张(occupyScale+1/级) · 改建地下室(unlock basement·暴力供奉) · 暗网摄影室(unlock av·AV系统) · 假山庭院(unlock courtyard·前置占据规模≥2) · AV设备升级(unlock av_advanced·前置摄影室)。

## 接入

- `EngineState` 加 upgrades账本 + 效果字段(totalSlots/purchaseUpgradeMult/turfFortifyBonus/occupyScale)。
- `economy.combatPower(avail, loyalty, upgradeBonus)` ×(1+bonus)；`settleDaily` 传 `combatBonus(upgrades)`。
- `useDayRunner` 用 `engine.totalSlots ?? opts.totalSlots`(行动格扩容生效) startDay/advanceToNextDay。

## 已确定决策（用户）
- 吞吐升级=堕落解档+花钱(吞吐扩容项+corruptionGate)。
- 打手3个NSFW项=纯NSFW联想+统一小战力(0.03/级)。
- 价格=每级固定价。
- 升级=子菜单非事件(v3 §0.1)。
