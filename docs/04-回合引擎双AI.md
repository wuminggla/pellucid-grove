# 04 · 回合引擎（双AI接入·engine）

> 模块：`src/game/engine/`。M5 产出。把 action-grid/paradigm/corruption/economy 串成"结算一个行动格"的流水线。
> AI 调用抽象成 AiPort（可注入），逻辑用 mock 单测；真实实现 real-ai 接底座 api-router。

## 这个模块做什么
**游戏的心脏**：玩家选一个行动格选项 → 引擎跑完整流水线 → 出正文 + 更新所有相关变量。
之前 M2-M4 是一堆独立纯函数，M5 把它们串成一条能跑的链。

## 核心：settleSlot(state, choice, opts)
```
1. pickParadigm        选范式 + 判是否首次特殊事件(查账本)
2. renderModeFor       决定 ai_full / ai_brief / fast_summary(快进)
3. 出正文:
   - fast_summary  → 不调AI,summaryTemplates填在场人数("大小姐给18人侍奉了")
   - ai_full/brief → 调 AI1 expand 扩写正文
4. extract(非快进)     调 AI2 抓叙事数值 → sanitizeExtract 防胡诌
5. corruption(仅首次)  gainCorruption→堕落度+认知防线+奖励闸门,发钱/打手,写首发账本
6. 返回 新state + 正文 + events
```

## 防胡诌：sanitizeExtract(raw, bounds)
- 原则(findings②,宽松)：只 ban 过于离谱的值(超 bounds)，不死抠。
- 越界/非数值字段被丢弃(保留旧值)，记入 rejected。
- **硬经营数值(资金/避孕套/威望/打手)不由AI抓**，由 economy 公式算——本引擎只处理"单格叙事数值(在场人数等) + 堕落 + 奖励闸门资源"。

## AiPort 抽象（双AI）
- `expand(req)`：AI1，按范式扩写正文。
- `extract(req)`：AI2，抓叙事数值(未校验,交 sanitize)。
- 测试注入 mock；真实用 `createRealAi({router, buildExpandMessages, buildExtractMessages})`：
  - expand 走 `router.call('story')`(主API)，抓 `<maintext>`。
  - extract 走 `router.call('vars')`(次API,未启用回落主API)，抓 `<vars>{...}</vars>` JSON。

## 关键设计决策
1. **AI 经 AiPort 注入**：编排逻辑与真实API解耦，可 mock 全链单测(不需API key)。
2. **首次特殊压过快进**：renderModeFor 保证堕落里程碑即使快进也调AI完整扩写。
3. **硬/软数值分治**：叙事数值AI抓+防胡诌；硬经营数值economy算。本引擎不碰资金/避孕套(除奖励闸门)。
4. **M5只做非流式**：一次性取完整响应。流式渲染(stream-parser)留UI层M7，不拖慢逻辑层。
5. EngineState 是状态切片(引擎实际读写字段)，完整 schema↔切片映射在 hook 层(M7)做。
