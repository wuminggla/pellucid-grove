# 09 · 强制事件接入（forced events）

> 落地 v3 蓝图 §4/§6.4/§10。把"系统扫描触发、非玩家主动选"的强制事件接进回合主流程。
> 三种插入强度：**强占**(锁一格) / **临时格**(预算外插一格) / **霸全**(夺整日)。

## 三种插入强度

| 强度 | 实现 | 用途 | 是否占预算 |
|------|------|------|-----------|
| 霸全 seize_day | `buildForcedLeaveDay` | 请假轮奸(欲望溢出夺次日) | 全日改造 |
| 强占 seize_slot | `lockSlot` | 地盘骚扰/火并防守 | 占用1个已有格 |
| 临时格 insert_slot | `insertEventSlot` | 避孕套归零(无空格也能插) | **不占**(预算外额外格) |

## action-grid 新增（machine.ts）

- **insertEventSlot(state, period, eventName, choice)**：在 cursor 后插入 `locked+inserted+planned` 临时格，不改 dayCount/nightCount/totalSlots（预算外），重排 index 使其下一格即执行。`ActionSlot.inserted` 标记。
- **buildForcedLeaveDay(dayNumber, totalSlots, serveChoice, eventName?)**：构造强制请假轮奸日——白天0格，全部格给夜晚=供奉、locked，phase 直接 day_settled（调用方 beginNight 即开始逐格供奉）。

## 霸全·请假轮奸（settlement + day-runner + hook）

- `EngineState.pendingForcedLeave`：夜晚欲望溢出信号，**随存档持久化**（跨天不丢）。
- `settleNight`：`overflow=true` → 置 `pendingForcedLeave`。
- `advanceToNextDay(engine, dayNumber, totalSlots, martial, infamy, serveChoice)`（纯函数）：每日结算 → 若 `pendingForcedLeave` 则 `buildForcedLeaveDay` 并清标记；否则 `startDay` 进玩家分配。返回 `{engine, day, daily, forcedLeave}`。
- `useDayRunner.nextDay` 调 `advanceToNextDay`；暴露 `forcedLeaveToday` 供 UI 提示。

## 强占 / 临时格（events + day-runner + hook）

- **ForcedEvent 扩展**（events/machine.ts）：加 `intensity('seize_slot'|'insert_slot')` + `optionId`(强占/插入的格跑哪个事件选项) + `label`。`condition` 改用 `ForcedContext`。
- **ForcedContext**（events/types.ts）= EventContext + `condomStock?` + `threatLevel?`（地盘系统未做先占位）。
- **forcedContextOf(engine)**（day-runner）：从 EngineState 构造 ForcedContext。
- **applyForcedInserts(day, engine, pool, period)**：扫 insert_slot 事件→`insertEventSlot`，标记 once 账本。在 `runCurrentSlot` 完成格【前】调用（cursor 仍指当前格→临时格成为下一格立即执行）。供奉归零即触发。
- **applyForcedSeizes(day, engine, pool)**：扫 seize_slot 事件→`lockSlot` 一格（白天优先，无白天格则夜晚）。在 `useDayRunner.allocate` 分配后调用。强占类一般非一次性（骚扰高频）。
- hook 暴露 `forcedInsert`(runCurrent返回) / `forcedSeize`(allocate后) 供 UI 提示。

## demo 数据（mock-ai.ts）

- `demoForcedPool`：①避孕套归零(insert_slot, once, 条件 condomStock≤0) ②地盘骚扰 harass(seize_slot, 条件 threatLevel≥1)。
- `demoEventOptions` 加 `condom_zero`(born_nsfw+首次里程碑) / `defend_turf`(born_sfw)。
- GameScreen 把 `demoForcedPool` 接进 settleOptions.forcedPool。
- **条件均为占位**：真实信号(库存/地盘威胁/稳定度)待经济/地盘系统驱动，机制已就位。

## 测试（116测）

- action-grid：insertEventSlot(cursor后插/不占预算/重排index/延后结算/无cursor追加) + buildForcedLeaveDay。
- settlement：溢出置 pendingForcedLeave / 未溢出不置。
- day-runner：advanceToNextDay(正常 vs 强制请假+清标记) / 供奉归零→插临时格+once账本 / 未归零不触发 / 骚扰强占(威胁触发锁格、无威胁不锁、请假日强占夜晚)。
