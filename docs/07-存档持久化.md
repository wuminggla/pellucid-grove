# 07 · 存档持久化（save）

> 模块：`src/game/save/` + SavePanel + useDayRunner.loadState。独立 IndexedDB(JiutiaoDB)。
> 数据安全铁律：读档不破坏现有存档；删除二次确认；深拷贝快照防引用污染；存档失败抛错不静默。

## 设计
- **独立数据库** JiutiaoDB（Dexie），与底座 SillyTavernWebDB 隔离，互不干扰。
- 一个存档 = RunnerState(day+engine) 快照 + fastForward + 摘要 + 元数据。
- DayState/EngineState 均为可序列化纯数据，直接 JSON 快照存储。

## 文件
- `save/types.ts`：GameSave / SaveSummary / SAVE_SCHEMA_VERSION
- `save/store.ts`：Dexie DB + putSave/autosave/listSaves/getSave/deleteSave/buildSummary
- `save/store.test.ts`：7 测（fake-indexeddb 真实往返）
- `components/game/SavePanel.tsx`：存档面板 UI

## 关键行为
- **新建存档**：自动分配槽位号，可命名。
- **覆盖存入**：复用 id，保留 createdAt 只更 updatedAt。
- **自动存档**：固定占 slot='auto' 单槽（覆盖式不累积）；GameScreen 在 dayNumber 变化(进新一天)时触发。
- **读档**：loadState 用快照恢复 day+engine+fastForward。
- **删除**：UI 二次确认（确认删除/取消），store.deleteSave 只删指定 id 不碰其它。
- **深拷贝快照**：putSave 内 JSON 深拷贝，存档后改原对象不污染存档（已测）。
- **时间戳由调用方传入**（now 参数），store 不内部依赖时钟，便于测试。

## UI（SavePanel）
- 顶部"💾 存档"按钮打开。
- 列出所有存档（auto 档金色标记⟳），摘要显示：第N天/堕落/认知/资金/打手/时间。
- 每档：读取 / 覆盖存入 / 删除(二次确认)。
- 新建存档输入框 + ＋按钮。

## 测试覆盖（数据安全重点）
- 往返完整性、深拷贝隔离、覆盖保留createdAt、autosave单槽不累积、列表倒序、删除不误伤其它档。
