# 05 · 行动格 UI（M7·游戏跑起来了）

> 模块：`src/components/game/` + `src/hooks/useDayRunner.ts` + `src/game/engine/day-runner.ts` + `mock-ai.ts`。
> M7 产出：把 M2-M5 的逻辑层接上 React UI，用 mock AI 在浏览器跑通完整一天循环（无需API key）。

## 这一步做了什么
游戏第一次"活"起来——能在浏览器里真正玩：分配8格 → 给每格选行动 → 逐格执行看结果 → 堕落/奖励闸门反馈 → 进次日。

## 新增文件
| 文件 | 作用 |
|------|------|
| `game/engine/day-runner.ts` | 连接 action-grid 状态机 与 settleSlot：runCurrentSlot = markRunning→settleSlot→completeCurrent，同步推进两份状态 |
| `game/engine/mock-ai.ts` | 假AI + 示例范式注册表(demoRegistry) + 总结词模板 + 防胡诌范围。供UI无API时跑 |
| `hooks/useDayRunner.ts` | React hook：管 RunnerState(day+engine)，命令式接口(allocate/setChoice/beginDay/runCurrent/nextDay/快进开关) |
| `components/game/StatusBar.tsx` | 状态栏：天数/资金/打手/堕落度/认知防线/在场/经期 |
| `components/game/GameScreen.tsx` | 主界面：分配菜单→白天执行→夜晚执行→次日；快进开关；结算反馈(首次特殊/奖励闸门) |

## 交互流程（GameScreen）
```
allocating  → 早7点分配菜单(白N/夜M按钮) → 每格下拉选行动(只列已解锁) → 确定分配
day_running → 逐格"执行当前格"(调mock AI出正文,当前格高亮"进行中") → 全做完→day_settled
day_settled → "进入夜晚"
night_running→ 逐格执行 → night_settled
night_settled→ "结束今天·进入次日"→ startDay(次日)
```

## 关键设计决策
1. **UI 不自己写游戏逻辑**：所有规则在 game/ 逻辑层(已测)，组件只调 hook。符合分层(表现→业务→数据)。
2. **mock AI 可换真AI**：GameScreen 用 createMockAi；接真API只需换成 createRealAi(router)，settleOptions 结构不变。
3. **下拉只列已解锁**：availableParadigms 过滤，未解锁选项(如肛交)不出现——解锁驱动内容展开。
4. **演示数据隔离**：demoRegistry/demoSummaryTemplates 在 mock-ai.ts，正式范式内容(接世界书)后续替换，不动UI。
5. App.tsx 直接渲染 GameScreen(替掉Vite默认页)。底座的通用 GameView(聊天式)保留备用，本游戏用专属 GameScreen。
