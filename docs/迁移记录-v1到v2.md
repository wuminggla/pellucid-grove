# 迁移记录 · v1(React+Vite+iframe) → v2(Vue+Webpack+jQuery.load)

> **日期**: 2026-06-27
> **决策者**: 用户(在 v1 试跑发现 iframe 被酒馆 sanitize 剥离后)
> **执行**: 一次性大迁移,业务代码零改动复用,UI 层 React→Vue 待翻译

## 为什么必须迁

### v1 设计(已弃)

```
酒馆 → 状态栏正则替换 <StatusPlaceHolderImpl/> 为 <iframe src="GitHub Pages URL">
        ↓
       iframe 加载 React app
        ↓
       postMessage 双向通信(自创协议)
```

### v1 试跑暴露的硬伤

| 问题 | 严重度 | 触发 |
|------|--------|------|
| **iframe 被酒馆 DOMPurify 剥离** | 致命 | F12 看不到 `<iframe>` 标签本体,只有空容器 div |
| jsdelivr 主域对 .html 返回 text/plain | 致命 | 浏览器把 React 应用当源码显示而非渲染 |
| IndexedDB origin 隔离在 jsdelivr.net | 重大 | 玩家间数据冲突·清浏览器数据丢失存档 |
| 自创 postMessage 协议跟酒馆生态不通 | 重大 | 用不上 `defineMvuDataStore` 等酒馆助手内置工具 |
| 跨楼层 iframe 状态不延续 | 中度 | 每条 AI 消息一个独立 iframe,内存 state 重置 |

### 用户决定 + 参考社区主流

用户指出"本地配置玩其他大前端卡都没问题"+ 提供两张参考卡截图说明"按钮+下载资源"模式。

查 `tavern-cards` skill 的 `references/packaging.md` 第 156-163 行,找到酒馆生态主流方案:

```html
<script>
$('body').load('https://testingcf.jsdelivr.net/gh/{USER}/{REPO}/dist/{Project}/界面/状态栏/index.html')
</script>
```

**核心区别**:
- 不用 iframe(绕过 sanitize 黑名单)
- 用 jQuery `$('elem').load(URL)` 同源 AJAX 拿 HTML 文本注入 DOM(脚本直接操作)
- 用 `testingcf.jsdelivr.net` 镜像(对 .html 返回 text/html 正确 MIME)
- 配合 [tavern_helper_template](https://github.com/StageDog/tavern_helper_template) 工程结构(Webpack 5 输出单文件 HTML,Vue/Pinia 通过 ESM CDN 在 runtime 加载)

## v2 设计

```
酒馆 → 状态栏正则替换 <StatusPlaceHolderImpl/> 为 <div id="pellucid-mount"> + <script>$.load(URL)</script>
        ↓
       jQuery 拿远程 HTML 注入 mount div(同源,绕过 sanitize)
        ↓
       Vue app 挂载到注入的 #app 容器
        ↓
       defineMvuDataStore(Schema) 双向绑定酒馆 MVU 变量(待实施)
```

### 部署机制变化

| | v1(已弃) | v2 |
|---|----------|------|
| 构建工具 | Vite | Webpack 5(模板自带) |
| 输出 | 多文件 dist + GitHub Pages | 单文件 HTML(JS/CSS 内联)+ dist 直接进 git |
| CDN | jsdelivr 拉 @gh-pages 分支 | jsdelivr 拉 master 分支 dist/ |
| 注入方式 | `<iframe src="...">` | `<div id="pellucid-mount">` + `$.load(URL)` |
| 镜像 | cdn.jsdelivr.net | **testingcf.jsdelivr.net**(MIME 友好) |
| GitHub Actions | 必需(build → push gh-pages) | 不需要(本地 build → commit dist → push) |
| 锁版本 | tag(失败,tag 不在 gh-pages) | commit sha(`@<sha7>`) |
