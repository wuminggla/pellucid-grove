# 01 · 行动格系统（action-grid）

> 模块：`src/game/action-grid/`。M2 产出。纯逻辑层（无 UI / 无 AI），可单测。
> 设计依据：经济模型_v1.md §13 运行机制定稿 + 运行时序蓝图（findings）。

## 这个模块做什么
管理"一天"的行动格生命周期：早7点分配 → 白天逐格执行 → 白天结算 → 夜晚逐格执行 → 夜晚结算。
是整个游戏的时序骨架。**只管流程状态，不管"格里发生什么"**（范式筛选/AI扩写/数值结算是 paradigm/economy/events 模块的事）。

## 文件
- `types.ts`：类型定义（DayPhase / SlotStatus / ActionSlot / DayState / Allocation / SlotChoice）。
- `machine.ts`：状态机纯函数（每个操作返回新 DayState，不改入参）。
- `machine.test.ts`：9 个单测，覆盖分配/请假/校验/强占/完整流转。

## 核心模型
- **一天 = 8格统一分配池**（totalSlots 来自 schema 时间.每日总行动格，可升级）。
- 玩家分配 `dayCount + nightCount = totalSlots`。
- **请假 = dayCount:0 全给夜晚**（合法极端分配，不需单独按钮）。
- 每格状态：`empty → planned → running → done`。
- **空格不能提交**（checkSubmit 校验）。
- **强占锁定**（lockSlot）：事件系统强占的格 locked=true，玩家不可改派/清空（对应强占/霸全事件）。

## 阶段流转（DayPhase）
```
allocating → (beginDay) → day_running → (逐格 markRunning→completeCurrent) → day_settled
          → (beginNight) → night_running → (逐格) → night_settled → [每日审查→次日]
白天0格(请假)：beginDay 直接跳 day_settled
```

## 对外主要接口（machine.ts）
- `startDay(dayNumber, totalSlots)` 开新一天
- `allocate(state, {dayCount,nightCount})` 分配，返回 {ok,error,state}
- `setChoice / clearChoice(state, period, index, choice)` 安排/清空选项
- `lockSlot(state, period, index, eventName, choice)` 事件强占
- `checkSubmit(state, period)` 提交校验（空格拦截）
- `beginDay / beginNight` 阶段流转
- `markRunning(state)` 标记当前格进行中（调AI/mock前）
- `completeCurrent(state, resultText)` 完成当前格+推进cursor，本时段完则结算
- `currentSlot(state)` 读当前执行格

## 关键设计决策
1. **纯函数 + 不可变**：所有操作返回新 state，便于 React 状态管理、时间旅行存档、单测。
2. **逐格 cursor 推进**：对应运行时序"玩家逐次点确定，当前格暗下显示进行中"。machine 只推进 cursor 与状态，实际调 AI 由上层（hook/业务）在 markRunning 与 completeCurrent 之间做。
3. **校验当前为"全排满才提交"**：设计稿提过"可只排第一格边走边排"，MVP 先要求全排满（checkSubmit）。若后续要边排边走，放宽 checkSubmit 为"至少首格非空"即可——已在代码注释标注。
4. **不管内容**：SlotChoice 只存 optionId+label+params，范式筛选/扩写/数值在别的模块，保持本模块单一职责。

## 待接入（后续模块对接点）
- markRunning→completeCurrent 之间：paradigm 选范式 → AI1 扩写 → AI2 抓数值 → economy 结算（M3/M4/M5）。
- lockSlot 的调用方：events 事件系统候选池命中强占/霸全事件时（M6）。
- night_settled 后：每日审查（金钱/威望硬失败检测）→ startDay 次日（endings/economy）。
- totalSlots 来源：schema 时间.每日总行动格（含设施升级"行动格扩容"）。
